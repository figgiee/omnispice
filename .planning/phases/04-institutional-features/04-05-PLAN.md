    STEP 1 — `src/labs/referenceRunner.ts`:
    ```typescript
    /**
     * Generate reference CSVs for all waveform_match checkpoints in a lab.
     * Drives the existing ngspice Web Worker (same path simulationStore uses)
     * against the instructor's reference circuit, extracts each probe vector,
     * emits a CSV, and uploads it to the Worker's reference endpoint.
     *
     * Transactional: all uploads must succeed OR nothing writes to D1.
     * On failure, surfaces a retry toast and leaves the lab in DRAFT state.
     */
    export async function generateAndUploadReferences(
      labId: string,
      lab: Lab,
      referenceCircuit: CircuitState,
      getToken: () => Promise<string | null>,
      onProgress?: (probe: string, status: 'running' | 'uploaded' | 'failed') => void,
    ): Promise<{ reference_waveform_keys: Record<string, string> }>;

    /**
     * Extract waveform_match probe names from every checkpoint in the lab.
     */
    export function collectProbes(lab: Lab): string[];

    /**
     * Serialize a (time, value) Float64Array pair to CSV text.
     */
    export function vectorsToCsv(time: Float64Array, values: Float64Array): string;
    ```

    Implementation notes:
    - Swap the active circuit via `circuitStore.setCircuit(referenceCircuit)`, snapshot the current circuit first so the editor can restore
    - Call `runSimulation` with the appropriate analysis (transient by default; authoring UX may let instructor specify)
    - Await `simulationStore.vectors` via a one-shot subscription or expose `runSimulationOnce(): Promise<VectorData[]>` helper
    - For each probe in `collectProbes(lab)`:
      - Find the vector by name (case-insensitive)
      - Build `{time, values}` Float64Arrays
      - CSV stringify
      - POST to `/api/labs/{labId}/reference/{probe}` with `Content-Type: text/csv`
    - Restore original circuit on completion or error
