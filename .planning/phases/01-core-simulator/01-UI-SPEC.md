# Phase 1 -- UI Design Contract

> Visual and interaction contract for the Core Simulator phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom CSS variables) |
| Preset | not applicable |
| Component library | custom (canvas-heavy app; React Flow + uPlot dominate the surface area) |
| Icon library | Lucide React (`lucide-react`) -- MIT, tree-shakeable, consistent 24x24 stroke icons |
| UI font | Inter (variable, `Inter Variable` via `@fontsource-variable/inter`) |
| Mono font | JetBrains Mono (variable, `@fontsource-variable/jetbrains-mono`) -- for component values, netlist text, measurement readouts |
| Styling approach | CSS Modules + CSS custom properties on `:root`. No Tailwind. No CSS-in-JS. |

**Rationale:** The application surface is 80%+ canvas (React Flow schematic editor) and chart (uPlot waveform viewer). shadcn and Tailwind add overhead without benefit when the primary rendering is SVG nodes, canvas pixels, and custom graph edges. A lean CSS variable system keeps the design contract tight and avoids class-name sprawl on canvas-rendered elements.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding, tight element pairs |
| sm | 8px | Button internal padding (vertical), list item horizontal padding, toolbar icon gaps, sidebar horizontal padding, input field padding, compact card padding |
| lg | 16px | Panel internal padding, section gaps within panels, form field spacing |
| xl | 24px | Between major panel sections, toolbar-to-canvas gap (none -- flush) |
| 2xl | 32px | Not used in Phase 1 (reserved for page-level layouts in Phase 2+) |
| 3xl | 48px | Not used in Phase 1 |

Exceptions:
- Toolbar height: 48px (12 * 4, accommodates 32px icon buttons + 8px vertical padding)
- Component library item height: 36px (touch-friendly row height)
- Canvas grid: 10px default (from D-02, not a multiple of 4 -- this is a schematic grid, not a UI spacing value)
- Panel drag handle hit area: 8px (invisible, overlaps 4px visible handle by 2px each side)

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Inter | 13px | 400 (regular) | 1.5 (20px) | Panel content, error messages, sidebar labels |
| Label | Inter | 11px | 600 (semibold) | 1.4 (16px) | Category headers, tab labels, toolbar labels, measurement labels |
| Heading | Inter | 16px | 600 (semibold) | 1.3 (21px) | Panel titles, dialog headings |
| Mono | JetBrains Mono | 12px | 400 (regular) | 1.5 (18px) | Component values ("10k"), netlist preview, measurement readouts, cursor values |

**Font weights used:** 400 (regular) and 600 (semibold) only. Label and Heading share 600; size difference (11px vs 16px) provides hierarchy.

**Font loading:** Variable fonts loaded via `@fontsource-variable`. System font stack as fallback: `Inter Variable, -apple-system, BlinkMacSystemFont, Segoe UI, sans-serif`. Mono fallback: `JetBrains Mono Variable, Consolas, Monaco, monospace`.

**Why 13px body:** This is a dense technical tool, not a reading application. 13px matches VS Code's default UI font size and Figma's panel text. It maximizes information density in the sidebar and error panels while remaining legible at 1.5 line height.

## Component Symbol Rendering

All symbols follow IEEE/IEC standard conventions (D-06). Rendered as SVG within React Flow custom nodes.

### Symbol Specifications

| Component | Symbol Style | Dimensions (SVG viewBox) | Pins |
|-----------|-------------|--------------------------|------|
| Resistor | Zigzag (US/IEEE) | 60 x 24 | 2 (left, right) |
| Capacitor | Two parallel plates | 40 x 32 | 2 (left, right) |
| Inductor | Loops/coils | 60 x 24 | 2 (left, right) |
| Diode | Triangle + bar | 40 x 32 | 2 (anode, cathode) |
| Zener Diode | Triangle + bent bar | 40 x 32 | 2 (anode, cathode) |
| NPN BJT | Circle with arrow-out emitter | 48 x 48 | 3 (base, collector, emitter) |
| PNP BJT | Circle with arrow-in emitter | 48 x 48 | 3 (base, collector, emitter) |
| NMOS | Standard MOSFET symbol | 48 x 48 | 3 (gate, drain, source) |
| PMOS | Standard MOSFET symbol with inversion bubble | 48 x 48 | 3 (gate, drain, source) |
| Op-Amp | Triangle with +/- inputs | 56 x 48 | 3 (non-inv, inv, output) -- power pins implicit |
| Voltage Source | Circle with +/- | 36 x 36 | 2 (positive, negative) |
| Current Source | Circle with arrow | 36 x 36 | 2 (in, out) |
| Ground | Standard 3-line ground | 24 x 24 | 1 (top) |
| Transformer | Two coupled inductors | 60 x 48 | 4 (pri+, pri-, sec+, sec-) |

### Symbol Rendering Rules

- Stroke width: 2px for all component outlines
- Stroke color: `--text-primary` (#e8eaf6)
- Fill: none (transparent) -- standard EE convention
- Pin dots: 4px radius circles at connection points, filled with `--text-primary`
- Component value label: positioned 8px below or to the right of the symbol, using Mono font at 12px
- Reference designator: positioned 8px above or to the left, using Body font at 11px, `--text-secondary`
- Rotation: 0, 90, 180, 270 degrees via R key (D-09). SVG transform-origin at center.
- Selected state: 2px accent-colored outline box around the full symbol + label area

## Interaction Contracts

### Right-Click Context Menu (Vision Ceiling — 2026-04-09)

> **Vision:** Right-clicking anywhere on the canvas feels like having a knowledgeable lab partner. Type two letters, the right component is already highlighted. No hunting the sidebar. Components ordered by: (1) recently used in this session, (2) electrically sensible at this circuit point, (3) alphabetical. Placing is instant — component appears at cursor, snapped to grid, pre-selected for immediate wiring. The quality bar is UE5 Blueprint node search + electrical intelligence that Blueprint can't have.
>
> **References:** UE5 Blueprint node search (interaction model), Figma quick-insert (cursor-placement feel), KiCad "A" key picker (EE-specific filtering precedent).

| Element | Value |
|---------|-------|
| Trigger | Right-click on empty canvas area |
| Position | Floating popup at cursor position, auto-repositions if near screen edge |
| Search input | Auto-focused on open, fuzzy match across all component names and aliases |
| Result ordering | (1) Recently used (last 5 placed), (2) context-aware suggestions, (3) alphabetical |
| Context awareness | If right-clicking near an unconnected pin, deprioritize components with incompatible port counts |
| Keyboard nav | Arrow keys move selection, Enter places at cursor-snapped position, Escape dismisses |
| Placement | Component placed at cursor position snapped to 10px grid, immediately selected |
| Pin right-click variant | Right-clicking a component pin shows "Start Wire" + filtered component search (same popup, pre-filtered to show components with compatible ports) |
| Dismiss | Escape, click outside, or any non-arrow key during idle |
| Max results shown | 8 (scrollable) |
| Empty state | "No components match — try 'res', '741', or 'ground'" |

## Animation and Transitions

| Element | Property | Duration | Easing | Notes |
|---------|----------|----------|--------|-------|
| Panel collapse/expand | height | 200ms | ease-out | Bottom panel and sidebar |
| Hover state change | background-color | 100ms | ease | Buttons, list items |
| Canvas pan-to-error | transform (viewport) | 200ms | ease-out | When clicking error to navigate (D-21) |
| Toast appear | opacity + translateY | 150ms | ease-out | Slides down from top-right |
| Toast dismiss | opacity | 150ms | ease-in | Fades out after 3s auto-dismiss |
| Dropdown open | opacity + scale | 100ms | ease-out | Analysis type dropdown |
| Wire connection snap | none | instant | -- | Magnetic snap is instant, no animation |
| Component placement | none | instant | -- | No bounce or scale animation on place |

**Principle:** Animations exist only for orientation (where did a panel go?) and state acknowledgment (toast appeared). Canvas interactions are always instant -- zero animation on place, connect, rotate, delete. Engineers want direct manipulation, not choreography.

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Focus management | Tab order: Sidebar search -> Sidebar items -> Toolbar buttons -> Canvas -> Bottom panel tabs -> Bottom panel content |
| Canvas keyboard nav | Arrow keys move selected component by 10px grid steps. Tab cycles through components. Enter activates value edit. |
| Screen reader | Canvas components announced as "{type} {ref} {value}" (e.g., "Resistor R1 10 kilohms"). Waveform data available as table via SR-only element. |
| Color contrast | All text passes WCAG AA on dark backgrounds. Smallest text (11px label) at `#9fa8c4` on `#16213e` = 5.2:1 ratio. |
| Motion | `prefers-reduced-motion`: disables panel slide animations, toast slide. Keeps instant canvas interactions (already no animation). |
| Focus visible | 2px `--accent-primary` outline offset by 2px on all focusable elements |
